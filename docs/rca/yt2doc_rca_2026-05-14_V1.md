# RCA - yt2doc YouTube Bot Detection Failure

- **日期**：2026-05-14
- **版本**：V1
- **關聯技能**：[yt2doc](.agents/skills/yt2doc/SKILL.md) / [ingest-youtube](.agents/skills/ingest-youtube/SKILL.md) / [daily-workflow](.agents/skills/daily-workflow/SKILL.md)

---

## 1. 觀察到的問題（Observed Problem）

在執行 `daily-workflow` 背景轉錄 YouTube 任務（影片 ID `55pTFVoclvE`，容器 ID `23eef63b-ccec-474b-96fc-767daf649815`）時，輪詢 `command_status` 發現容器以 Exit Code 1 異常終止。錯誤輸出顯示：
```
yt_dlp.utils.DownloadError: ERROR: [youtube] 55pTFVoclvE: Sign in to confirm you’re not a bot. Use --cookies-from-browser or --cookies for the authentication. See https://github.com/yt-dlp/yt-dlp/wiki/FAQ#how-do-i-pass-cookies-to-yt-dlp for how to manually pass cookies.
```
轉錄下載被 YouTube 防爬蟲機制攔截，無法取得音訊檔進行後續轉錄。

---

## 2. 根因分析（Root Cause Analysis）

### 2.1 假說驗證（Hypotheses & Evidence）

- **假說 A：影片為不公開（Private）或已遭上傳者刪除。**
  - **驗證證據**：根據錯誤訊息，提示為 `Sign in to confirm you’re not a bot`，而非 `Private video` 或 `Video unavailable`。
  - **結論**：**拒絕假說 A**。

- **假說 B：YouTube 伺服器端加強了對機器人與伺服器 IP 的反爬蟲與機器人檢測（Bot Detection）機制，導致未夾帶使用者身分 Cookie 的 `yt-dlp` 下載請求遭封鎖。**
  - **驗證證據**：`yt2doc` 容器底層使用 `yt-dlp` 進行音訊提取，在預設情況下不具備登入狀態或瀏覽器 cookies。近期 YouTube 頻繁針對無頭（Headless）腳本與非瀏覽器特徵的流量發起驗證碼攔截或機器人檢測。
  - **結論**：**接受假說 B (主要根因)**。

### 2.2 根因結論

- **主要根因**：YouTube 反爬蟲與機器人檢測機制升級。底層 `yt-dlp` 在缺乏使用者 Cookie 的情況下觸發了安全防禦攔截。
- **次要因素**：`yt2doc` 容器未提供自動匯入或維護 YouTube Cookies 的自動化機制，在遭遇攔截時無法自動恢復。

---

## 3. 採取的修復（Fix Applied）

1. **攔截與通報**：依據 `ingest-youtube/SKILL.md` 規範，當容器結束代碼非零時，不將任務標記為已完成，保留 Google Task 於代辦清單中。
2. **通報使用者**：向使用者回報具體的錯誤訊息，提示後續需透過導出瀏覽器 cookies（例如傳遞 `--cookies` 參數）或手動轉錄處理。
3. **建立 RCA 紀錄**：將此問題與根因紀錄於 RCA 文件，以供未來優化 `yt2doc` 技能時參考。
