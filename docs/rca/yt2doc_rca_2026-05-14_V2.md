# RCA - yt2doc ModuleNotFoundError (torchvision)

- **日期**：2026-05-14
- **版本**：V2
- **關聯技能**：[yt2doc SKILL.md](.agents/skills/yt2doc/SKILL.md) / [yt2doc README.md](.agents/skills/yt2doc/README.md)

---

## 1. 觀察到的問題（Observed Problem）

在執行 `docker run` 測試 YouTube 影片轉錄（影片 ID `55pTFVoclvE`），以驗證 YouTube 機器人檢測防護問題是否已解決時，容器在啟動後隨即以 Exit Code 1 異常終止。根據終端機標準錯誤輸出（stderr），出現以下模組匯入錯誤：
```
  File "/root/.local/share/uv/tools/yt2doc/lib/python3.12/site-packages/transformers/models/aria/image_processing_aria.py", line 21, in <module>
    from torchvision.transforms.v2 import functional as tvF
ModuleNotFoundError: No module named 'torchvision'
```
這導致 `yt2doc` 無法執行任何後續的影片下載或轉錄處理。

---

## 2. 根因分析（Root Cause Analysis）

### 2.1 假說驗證（Hypotheses & Evidence）

- **假說 A：主機端掛載目錄參數錯誤或網路連線中斷。**
  - **驗證證據**：終端機日誌顯示錯誤源自於容器內部的 Python 套件路徑 `/root/.local/share/uv/tools/yt2doc/...` 中的 `importlib` 載入失敗，屬於純粹的 Python 套件依賴缺失問題，與外部網路連線或主機端掛載目錄無關。
  - **結論**：**拒絕假說 A**。

- **假說 B：`<your-dockerhub>/yt2doc` Docker 映像檔建置時套件依賴不完整。`transformers` 套件嘗試匯入 `torchvision`，但容器環境中並未安裝該套件。**
  - **驗證證據**：依據量化的錯誤日誌輸出，明確指出 `ModuleNotFoundError: No module named 'torchvision'`。當 `yt2doc` 啟動並初始化相關 AI 模組時觸發此依賴錯誤。
  - **結論**：**接受假說 B (主要根因)**。

### 2.2 根因結論

- **主要根因**：Docker 映像檔 (`<your-dockerhub>/yt2doc`) 內部的 Python 虛擬環境套件依賴損毀或缺失 (`torchvision`)，導致程式無法完成套件初始化。
- **促成因素**：容器建置或套件封裝流程未充分涵蓋所有執行時期所需的依賴套件。

---

## 3. 採取的修復（Fix Applied）

1. **問題定位與記錄**：完整擷取並分析日誌輸出，建立此 V2 版本的 RCA 文件。
2. **通報使用者**：明確向使用者回報無法驗證機器人攔截問題（因遭遇底層套件依賴崩潰），並提供具體錯誤訊息以利映像檔維護者後續重構建置環境。
